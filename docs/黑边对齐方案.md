# xian 黑边对齐方案

> **问题**:rsimg(卫星影像)边缘有 no-data 黑边,但 rn graph(路网)覆盖更大,黑边区仍有路。
> **目标**:对齐 rn 和 rsimg 覆盖——黑边区(无影像)的 rn 也要删掉对应路网。

---

## 三种方案

### 方案A:按黑边裁 rn graph(推荐)

**做法**:对每个 region,检测 sat 黑边区,把落在黑边区内的 rn 节点/边删掉。

```
对每个 region:
  1. 读 sat,算黑边 mask (sat.max(axis=2) < 15)
  2. 读 rn graph (.p),对每条边的两个端点:
     - 若端点落在黑边区 → 删该节点 + 连带边
     - 若边的中间穿过黑边区(端点不黑但中间黑)→ 裁断,只留非黑边部分
  3. 保存裁后的 .p,重新渲染 road_mask/gt.png/partial.png
```

**优点**:rn 和 sat 像素级对齐,彻底解决
**缺点**:边的中间穿过黑边要裁断(clip),稍复杂;需重新生成所有标签

### 方案B:直接删边缘 region(最简单)

**做法**:从 data_split.json 删掉 38 个边缘 region(或只删 train 的 29 个)。

**优点**:1 行改 data_split,无代码
**缺点**:
- test 的 5 个删了 → 和之前结果不可比(但之前结果本来就含失真,重测更公平)
- 损失 10% 数据(train 29 个)
- 治标不治本(剩余 region 仍可能有少量黑边)

### 方案C:黑边 mask 掉 sat + rn(双方对齐到"无数据")

**做法**:检测黑边区,把 rn 在黑边区的路也抹掉(rn 渲染时跳过黑边像素),sat 保持黑边。

```
对每个 region:
  1. 算黑边 mask
  2. 渲染 road_mask/gt.png/partial.png 时,黑边区强制置 0(无路)
  3. rn graph .p 不改(只改渲染的 mask),或同步删黑边区节点
```

**优点**:不动 graph .p,只改 mask 渲染
**缺点**:graph .p 和 mask 不一致(.p 有黑边区节点但 mask 没有),后续可能混乱

---

## 推荐:方案A

最彻底,rn graph 和 sat 像素级对齐。实现:
- 写脚本 `tools/clip_rn_to_sat.py`
- 对每个 region:检测黑边 → clip rn graph(.p)→ 重新生成 road_mask/gt.png/partial.png
- 重新生成所有标签后,重训 sam_road

边裁断处理:
- 端点在黑边 → 删节点 + 连带边
- 端点都不在黑边但边穿过黑边 → 用 cohen-sutherland 裁剪(已有实现 graph_ops.py L124),保留非黑边段,黑边段删

## 影响范围

改完需重新生成:
- `region_{c}_refine_gt_graph.p`(clip 后的 rn)
- `road_mask_{c}.png` / `keypoint_mask_{c}.png`(从 clip 后 rn 渲染)
- `region_{c}_gt.png`(从 clip 后 rn 渲染)
- `region_{c}_refine_gt_graph_partial.p` / `.png`(从 clip 后 rn 采样)

然后重训 sam_road(extraction + completion)。
