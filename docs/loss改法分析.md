# Loss 改法实验结果与反思

> **背景**:component 采样(不破碎partial)让 completion 从 0.588 降到 0.457。假设根因是"known edge 不参与 loss,全 known patch 无监督",试方案1(known edge 标签True参与loss)。

---

## 一、方案1 实验结果(didi_xian)

| 版本 | 采样 | loss | APLS | TOPO F1 | P | R |
|------|------|------|:---:|:---:|:---:|:---:|
| 0625 | 按边随机 | 旧(known排除) | **0.5878** | 0.7925 | 0.856 | 0.738 |
| 0626 | component | 旧(known排除) | 0.4571 | 0.6316 | — | — |
| 0626_1215 | component | **方案1(known标签True)** | **0.3910** | 0.6167 | 0.745 | 0.526 |

**方案1 更差了**(0.457→0.391),且:
- precision 降(0.856→0.745):known edge True 标签主导,模型偏向输出 True ✓ 符合预警
- **recall 也降**(0.738→0.526):模型依赖先验输出 True,未知边没先验反而不敢预测 True → 丧失视觉判断

APLS 不受阈值影响,0.391 是真实退化,不是阈值假象(阈值已重新标定 TOPO_THRESHOLD=0.596)。

---

## 二、方案1 为什么失败:捷径监督

**known edge 的 True 标签成了捷径**,和 traj 捷径问题本质相同:
- known edge 通过 road_feature_map + GNN 注入
- 模型学到"有先验 = 输出 True"
- 未知边没有先验,模型不敢预测 True → recall 降
- 即:known edge 监督破坏了未知边的纯视觉判断能力

**核心矛盾**:
- 旧 loss(known 排除):component 时全 known patch 无监督 → 退化
- 方案1(known 标签True):捷径监督,破坏未知边视觉判断 → 更差
- 两条路都不对

---

## 三、重新审视:问题可能在采样策略,不在 loss

### 3.1 component 采样的本质问题
component 让 known edge 成块聚集。这改变了 GNN 看到的拓扑结构:
- 按边随机:known edge 散布,GNN 看到的是"点状已知+点状未知"混合拓扑
- component:known edge 成块,GNN 看到的是"片状已知+片状未知"分离拓扑

GNN 在散布拓扑上学的消息传递模式,可能比成块拓扑更易泛化到未知边。

### 3.2 completion 任务对 partial 形态的隐含偏好
当前架构(SAM视觉 + GNN拓扑 + TopoNet)可能**天然偏好散布的 known edge**:
- 散布:每个局部都有已知锚点,GNN 能从已知推断邻近未知
- 成块:大片无已知区域,GNN 消息传递到不了,无法推断

这和"真实场景 partial 不破碎"矛盾。真实已知路网确实成块(整片已测绘),但当前架构处理不了成块形态。

---

## 四、可选方向

### 方向A:回退按边随机(保效果,承认 component 不 work)
- 训练+推理都用按边随机(0625 状态)
- 效果恢复 0.588/0.70
- 代价:partial 破碎(不真实),但架构能处理

### 方向B:方案2(known edge 降权 0.3)再试一次
- known edge 参与 loss 但权重低,避免主导
- 可能避免捷径(权重低,模型不能只靠 known edge)
- 风险:仍可能破坏未知边视觉判断(即使降权,捷径仍存在)

### 方向C:接受 partial 破碎是"架构需求"
- 论文叙事:partial 采样策略服务于架构,破碎 partial 让 GNN 消息传递更有效
- 不追求"真实不破碎",而是"架构友好"
- 真实场景的 partial 可以预处理(把成块已知打散成散布?不合理)

### 方向D:改架构让 GNN 能处理成块拓扑
- 长程依赖(全连接注意力替代局部 GNN 消息传递)
- 让 GNN 能跨大片无已知区域传播信息
- 改动大,研究风险高

### 方向E:混合采样(训练散布,推理成块)
- 训练用按边随机(架构友好,监督均匀)
- 推理用 component(真实不破碎)
- train/infer mismatch,但模型见过散布 known,推理成块可能仍能处理
- 需实验验证

---

## 五、建议

**优先方向A(回退按边随机)**,理由:
1. 方案1 失败证明"改 loss 让 component work"这条路难走(捷径问题)
2. 0625 按边随机已验证有效(0.588/0.70)
3. component 在当前架构下可能本质不 work(GNN 拓扑偏好散布)
4. 论文可叙事为"采样策略匹配架构"(方向C)

**若想再试一次**:方向B(方案2降权)或方向E(训练散布推理成块),但预期收益不确定。

**长期**:方向D(改架构)是治本,但工作量大。
