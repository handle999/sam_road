# 实验总结与踩坑记录

> SAM-Road Completion 全程实验汇总。仅列结论与坑,不复述代码。

---

## 〇、任务设定

**多模态互补的路网补全(Road Network Completion)**

核心思想:**traj 是训练增强模态,不是推理输入**。

| 阶段 | traj | rn(已知路网) | 说明 |
|---|---|---|---|
| **训练** | ✅ 用 | ✅ 用(edge_random 随机采样 partial) | traj 让模型从轨迹学到路网形态,增强视觉特征;rn 用按边随机采样(训练稳定) |
| **推理** | ❌ 不用 | ✅ 用(component 不破碎 partial) | traj 可达性比 rsimg 差太多+处理复杂,实际场景不靠谱;任务从 extraction 变 completion |

**相比原版 SAM-Road 的转变**:推理从纯视觉 extraction 变成基于已知 rn 的 completion。traj 的价值在训练时增强,推理时省掉。

**要验证的东西**:
1. **traj 训练增强的价值**:有 traj 训练 vs 无 traj 训练(方案A vs B),同样 img+rn 推理
2. **rn 推理补全的价值**:traj 训练的模型,推理加 rn vs 不加 rn
3. **退化正确性**:completion 无先验(无traj无rn)≈ extraction,不劣化
4. **partial 策略**:edge_random 训练 + component 推理(训练稳定 + 真实不破碎)
5. **黑边对齐**:xian 裁剪对齐 sat 黑边后重训,结果更干净

**数据集**:
- **DiDi Xian**:有 traj(快递员 GPS),可测全部目标
- **SpaceNet**:无 traj,只能测 extraction vs completion(rn-only)

## 一、最终结果(xian 黑边裁剪对齐后,2026-06-28)

### 1.1 didi_xian(traj 训练,ep9 ckpt,推理用 component partial)

| 档 | 训练 | 推理 | partial | APLS | TOPO F1 | P | R |
|---|---|---|---|:---:|:---:|:---:|:---:|
| extraction 基线 | img | img | — | 0.2183 | 0.5219 | 0.875 | 0.372 |
| 目标1(无traj无rn) | img+traj | img | — | 0.4541 | 0.6164 | 0.869 | 0.478 |
| trajonly(参考,偏离设定) | img+traj | img+traj | — | 0.4753 | 0.6359 | 0.892 | 0.494 |
| rn-only(**主设定**) | img+traj | img+rn | component | **0.5808** | 0.7460 | 0.845 | 0.668 |
| full(目标3) | img+traj | img+traj+rn | component | **0.6099** | **0.7573** | 0.861 | 0.676 |
| ~~rn-only(edge_random)~~ | img+traj | img+rn | edge_random | 0.5829 | 0.7575 | 0.840 | 0.690 |
| ~~full(edge_random)~~ | img+traj | img+traj+rn | edge_random | 0.5864 | 0.7724 | 0.861 | 0.700 |

> - **主设定 = rn-only(component)**:traj 训练增强 + rn 推理补全,不用 traj 推理。APLS 0.5808
> - full = 推理额外带 traj(偏离设定,但看 traj 推理增量)。APLS 0.6099
> - edge_random 行(删除线)= 推理用 edge_random partial,作对比参考,非主设定
> - extraction APLS 0.2183 异常低(裁剪后 road_mask 变化 + extraction 没用 rn),见附录分析

### 1.2 核心结论(didi_xian)

1. **rn 推理补全价值巨大**:目标1(0.4541)→ rn-only(0.5808),APLS +0.127(+28%)。rn 是 completion 的核心先验
2. **traj 推理增量小且偏离设定**:rn-only(0.5808)→ full(0.6099),+0.029。traj 推理不是设定方向
3. **退化基本正确**:目标1(0.4541)> extraction(0.2183),completion 模型无先验时不劣于 extraction(甚至更好,因 traj 训练增强)
4. **component vs edge_random 推理**:rn-only 下 component(0.5808)≈ edge_random(0.5829),差距极小(-0.002)。full 下 component(0.6099)> edge_random(0.5864),component 反而更好(+0.024)
5. **extraction APLS 0.2183 异常低**:裁剪后 road_mask 变化影响 extraction(纯视觉,无 rn 补偿),见附录

## 二、partial 采样策略实验

### 2.1 训练采样 vs 推理采样(核心发现)

| 训练采样 | 推理 partial | APLS | TOPO F1 | val_loss |
|---|---|:---:|:---:|---|
| 按边随机 | edge_random | 0.5864 | 0.7724 | 平滑 |
| 按边随机 | component | 0.6099 | 0.7573 | 平滑 |
| component | component | 0.4571 | 0.6316 | **震荡** |

**结论**:按边随机训练 + component 推理(0.6099)**远好于** component 训练(0.4571)。
- 问题在 component **训练**的震荡(每 epoch 整块留/删,训练目标波动)
- 模型在散布 partial 上稳定训练后,能泛化到成块 partial
- **最终方案:训练 edge_random(稳定),推理 component(真实不破碎)**

### 2.2 spacenet(无 traj)

| 训练采样 | 推理 | APLS | TOPO F1 |
|---|---|:---:|:---:|
| 按边随机 | img+rn | 0.7000 | 0.7964 |
| component | img+rn | 0.5293 | 0.7009 |

> spacenet 同样:component 训练退化(0.70→0.53)。

## 三、踩坑记录

### 坑1:坐标系 bug(completion 比 extraction 差的根因)
- **症状**:completion APLS 0.337 < extraction 0.375,给先验反而更差
- **根因**:known graph pickle 是 `(row,col)`,但 render/匹配/注入期待 `(x,y)`,漏了 coord_transform。didi_xian 是 x/y 互换,spacenet 还要 y 翻转
- **验证**:边端点正确率 0%→100%,render IoU 0.04→0.59
- **修复**:两个入口(`get_known_adj_for_render` + `load_known_graph`)统一转 `(x,y)`

### 坑2:component 采样训练震荡
- **症状**:component 训练后 APLS 0.588→0.457,val_loss 0.2~1.5 震荡
- **根因**:component 每 epoch 整块留/删,训练目标波动
- **修复**:训练回退 edge_random;推理用 component

### 坑3:方案1 loss(known edge 标签 True)更差
- **症状**:APLS 0.457→0.391,precision/recall 双降
- **根因**:known edge 标签 True 成捷径监督,破坏纯视觉判断
- **修复**:撤销,回退旧 loss(known edge 排除出 loss)

### 坑4:spacenet rn 增量为零
- **根因**:extraction recall 0.69(强基线),rn 提供的已知边大部分已提全 → rn 冗余
- **结论**:rn 价值与 extraction 基线强度负相关

### 坑5:partial 可复现性
- **根因**:全局 random 在多文件间累积
- **修复**:per-file `random.Random(42)`

### 坑6:Dropout 步 valid 未恢复
- **修复**:dropout 步把真实候选对 valid 恢复 True

### 坑7:推理 dtype bug
- **修复**:batch_points 加 `dtype=torch.float32`

### 坑8:任务设定理解偏差(traj 角色)
- **纠正**:traj 是训练增强模态,推理不用。正确消融是"训练有无 traj"

### 坑9:xian 黑边不匹配
- **症状**:38 个边缘 region sat 有黑边但 rn 有路(rn 覆盖 > sat 覆盖)
- **修复**:cohen-sutherland 裁剪 rn graph 对齐 sat,裁断处补 node。裁后黑边区路 35.9%→0.16%

### 坑10:partial PNG thickness 不一致
- **修复**:partial.png thickness 3→2,对齐 prepare_dataset 的 gt.png 和 DelvMap(都 2)

## 四、最终方案

| 场景 | 采样 | 说明 |
|---|---|---|
| 训练 | edge_random | epoch 间稳定,监督均匀 |
| 推理 | component | 真实不破碎,edge_random 训练可泛化 |

**核心权衡**:edge_random partial 破碎(不真实)但训练稳定;component 不破碎(真实)但训练震荡。解法:**训练 edge_random 保稳定,推理 component 选真实**。
