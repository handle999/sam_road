# 实验总结与踩坑记录

> SAM-Road Completion 全程实验汇总。仅列结论与坑,不复述代码。

---

## 〇、任务设定(修正)

**多模态互补的路网补全**。核心:traj 是**训练增强模态**,不是推理输入。

| 阶段 | traj | rn | 说明 |
|---|---|---|---|
| 训练 | ✅ 用 | ✅ 用(随机 partial) | traj 增强 vision 理解(轨迹=路),让模型学到更鲁棒视觉特征 |
| 推理 | ❌ 不用 | ✅ 用(真实已知路网) | traj 可达性比 rsimg 差太多+处理复杂,实际场景不靠谱 |

**任务转变**:相比原版 SAM-Road(extraction 纯视觉),推理任务变成 completion(基于已知 rn 补全)。traj 的价值在训练时增强,推理时省掉。

**正确消融**:
- A:无 traj 训练 + img+rn 推理(baseline,待补)
- **B:traj 训练 + img+rn 推理(本方案,0.6226)**
- C:traj 训练 + img 推理(无 rn,验证 rn 推理价值,0.4332)

> 之前做的 trajonly/rnonly 推理消融(traj 当推理输入)偏离设定,但仍保留作参考——见 §一。

## 二、partial 采样策略实验(完整指标)

### 2.1 didi_xian(traj 训练,ep9 ckpt,统一阈值 ITSC0.059/ROAD0.179/TOPO0.596)

| 训练采样 | 推理 partial | APLS | TOPO F1 | P | R | val_loss |
|---|---|:---:|:---:|:---:|:---:|---|
| 按边随机 | 按边随机(per-file seed42) | **0.6226** | **0.7833** | 0.850 | **0.726** | 平滑 |
| 按边随机 | component(不破碎) | 0.5984 | 0.7550 | **0.863** | 0.671 | 平滑 |
| component | component | 0.4571 | 0.6316 | 0.690 | 0.582 | **震荡** |
| component | component+方案1loss | 0.3910 | 0.6167 | 0.745 | 0.526 | **震荡** |

**关键发现**:
- 按边随机训练 + component 推理(0.5984)**远好于** component 训练(0.4571),+0.141
- component 训练 val_loss 震荡(0.2~1.5),按边随机训练平滑收敛
- component 推理比按边随机推理:precision 升(+0.013,更保守),recall 降(-0.055,大片无已知区漏检)
- 方案1 loss(known 标签True)更差:捷径监督,recall 降

### 2.2 spacenet(无 traj)

| 训练采样 | 推理 | APLS | TOPO F1 | P | R |
|---|---|:---:|:---:|:---:|:---:|
| 按边随机 | img+rn | **0.7000** | 0.7964 | 0.931 | 0.696 |
| component | img+rn | 0.5293 | 0.7009 | 0.965 | 0.551 |

> spacenet 同样:component 训练退化(0.70→0.53),recall 大降(0.696→0.551)。

### 2.3 训练采样 vs 推理采样的解耦

| | 训练震荡? | 推理效果 |
|---|---|---|
| 按边随机训练 | ❌ 平滑 | 推理按边随机最好(0.6226),推理 component 也可(0.5984) |
| component 训练 | ✅ 震荡 | 推理 component 差(0.4571) |

**结论**:问题在 component **训练**的震荡(每 epoch 整块留/删,训练目标波动),不在 component partial 本身。模型在散布 partial 上稳定训练后,能泛化到成块 partial。

## 三、踩坑记录

### 坑1:坐标系 bug(completion 比 extraction 差的根因)
- **症状**:completion APLS 0.337 < extraction 0.375,给先验反而更差
- **根因**:known graph pickle 是 `(row,col)`,但 render/匹配/注入期待 `(x,y)`,漏了 coord_transform。didi_xian 是 x/y 互换,spacenet 还要 y 翻转
- **验证**:边端点正确率 0%→100%,render IoU 0.04→0.59
- **修复**:两个入口(`get_known_adj_for_render` + `load_known_graph`)统一转 `(x,y)`,下游不动
- **教训**:训练侧坐标系对(经 coord_transform),bug 只在推理。extraction 不受影响(无 known graph 路径)

### 坑2:component 采样训练震荡
- **症状**:component 采样训练后 didi APLS 0.588→0.457,spacenet 0.70→0.53,val_loss 0.2~1.5 震荡
- **根因**:component 每 epoch 整块留/删,known edge 集合剧烈变化,训练目标波动。按边随机每 epoch 只小幅变化,训练稳定
- **验证**:component+旧loss 和 component+方案1loss 都震荡 → 震荡根源是 component 采样,非 loss
- **修复**:训练回退按边随机;推理可选 component(按边随机训练+component推理 0.5984)
- **教训**:采样策略的"epoch 间方差"比"形态真实性"更重要。component 适合固定 partial(推理),不适合每 epoch refresh(训练)

### 坑3:方案1 loss(known edge 标签 True)更差
- **症状**:component + 方案1 loss,APLS 0.457→0.391,precision/recall 双降
- **根因**:known edge 标签 True 成捷径监督,模型依赖先验输出 True,未知边没先验不敢预测 True → recall 降(同 traj 捷径问题)
- **修复**:撤销方案1,回退旧 loss(known edge 排除出 loss)
- **教训**:known edge 通过 road_feature_map+GNN 注入,"标签 True"会变成"有先验=True"的捷径,破坏纯视觉判断。改 loss 救不了采样问题

### 坑4:spacenet rn 增量为零
- **症状**:spacenet rn-only(0.7000)≈ extraction(0.7012),rn 没增益
- **根因**:extraction recall 0.69(强基线),rn 提供的 50% 已知边大部分 extraction 已提全 → rn 冗余。didi_xian extraction recall 0.44(弱基线),rn 真补漏 → +0.138
- **结论**:rn 价值与 extraction 基线强度负相关。不是 rn 无效,是 spacenet extraction 已强
- **教训**:completion 卖点聚焦"弱基线/复杂场景",不在强基线上硬拼

### 坑5:partial 可复现性
- **症状**:同 seed 42 两次跑 partial 结果不同
- **根因**:全局 random 在多文件间累积,文件顺序/空图跳过导致随机流偏移
- **修复**:每张图独立 `random.Random(42)`,采样只依赖 seed+该图
- **教训**:多文件采样用 per-file seed,不依赖全局 random 流

### 坑6:Dropout 步 valid 未恢复(P0-1)
- **症状**:模态 dropout 清零先验后,valid 掩码没恢复,20% 训练步无先验+只监督半数边
- **根因**:`sample_patch` 在 dropout 前算好 valid(known edge=False),dropout 后没更新
- **修复**:dropout 步把真实候选对 valid 恢复 True
- **教训**:dropout 改变输入分布时,下游掩码要同步更新

### 坑7:推理 dtype bug
- **症状**:CPU 推理报 `expected Float but found Double`
- **根因**:`batch_points = torch.tensor(collated['points'])` 未指定 float32,numpy float64 传入
- **修复**:三个 inferencer 的 batch_points 加 `dtype=torch.float32`
- **教训**:numpy→torch 转换显式指定 dtype

### 坑8:任务设定理解偏差(traj 角色)
- **症状**:做了 trajonly/rnonly 推理消融(traj 当推理输入),偏离设定
- **纠正**:traj 是训练增强模态,推理不用。正确消融是"训练有无 traj",不是"推理有无 traj"
- **待补**:无 traj 训练的 completion ckpt(方案A),对比 traj 训练(方案B 0.6226)验证 traj 训练增强价值

## 四、最终方案

| 场景 | 采样 | 说明 |
|---|---|---|
| 训练 | 按边随机 | epoch 间稳定,监督均匀 |
| 推理(默认) | 按边随机 | 与训练一致,效果最好(didi 0.6226) |
| 推理(真实不破碎) | component | 按边随机训练+component推理(didi 0.5984),可接受 |

**核心权衡**:按边随机 partial 破碎(不真实)但训练稳定;component 不破碎(真实)但训练震荡。解法:**训练用散布保稳定,推理按需选形态**。
