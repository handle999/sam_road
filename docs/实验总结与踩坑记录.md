## 〇、任务设定

**多模态互补的路网补全(Road Network Completion)**

核心思想:**traj 是训练增强模态,不是推理输入**。

| 阶段 | traj | rn(已知路网) | 说明 |
|---|---|---|---|
| **训练** | ✅ 用 | ✅ 用(edge_random 随机采样 partial) | traj 让模型从轨迹学到路网形态,增强视觉特征;rn 用按边随机采样(训练稳定) |
| **推理** | ❌ 不用 | ✅ 用(component 不破碎 partial) | traj 可达性比 rsimg 差太多+处理复杂,实际场景不靠谱;任务从 extraction 变 completion |

**相比原版 SAM-Road 的转变**:推理从纯视觉 extraction 变成基于已知 rn 的 completion。traj 的价值在训练时增强,推理时省掉。

**要验证的东西**:
1. **traj 训练增强的价值**:有 traj 训练 vs 无 traj 训练(方案A vs B),同样 img+rn 推理
2. **rn 推理补全的价值**:traj 训练的模型,推理加 rn vs 不加 rn
3. **退化正确性**:completion 无先验(无traj无rn)≈ extraction,不劣化
4. **partial 策略**:edge_random 训练 + component 推理(训练稳定 + 真实不破碎)
5. **黑边对齐**:xian 裁剪对齐 sat 黑边后,train/eval/推理 GT 统一

**数据集**:
- **DiDi Xian**:有 traj(快递员 GPS),378 region(裁剪对齐后),可测全部目标
- **SpaceNet**:无 traj,只能测 extraction vs completion(rn-only)

### 1.3 核心结论

#### sam_road 内部消融
1. **rn 推理补全价值巨大**:目标1(0.4667)→ rn-only(0.5965),APLS +0.130(+28%)。rn 是 completion 的核心先验
2. **traj 推理增量小且偏离设定**:rn-only(0.5965)→ full(0.6204),+0.024。traj 推理不是设定方向
3. **退化正确**:目标1(0.4667)> extraction(0.4211),completion 模型无先验时不劣于 extraction(traj 训练增强)
4. **recall 提升是 rn 的核心贡献**:extraction R=0.4591 → rn-only R=0.6909(+0.232),rn 大幅补全漏检路;precision 略降(0.895→0.845)但 F1 净赚

#### sam_road vs P2CNet(公平对比,50% fixed)
5. **APLS:P2CNet(0.6720)> sam_road rn-only(0.5965)**,但 < sam_road full(0.6204)差距缩小。P2CNet 像素级补全 + A* 连边,连通性由 mask 保证;sam_road TopoNet 逐边预测易断边
6. **TOPO F1:sam_road(0.7602/0.7714)≫ P2CNet(0.6697)**。sam_road graph 原生输出,拓扑精确性显著更高(+0.09~0.10)
7. **P/R 权衡**:P2CNet 极高 precision(0.960)低 recall(0.514,保守漏检);sam_road 更均衡(P≈0.85, R≈0.69),recall 远高于 P2CNet(+0.18)
8. **评测口径差异**:P2CNet mask→骨架提图(有转换损失,mask 断裂处图断路);sam_road graph 原生(无转换)
9. **范式差异**:P2CNet 像素级补全在连通性(APLS)上占优(mask 连续→A* 保证连通),sam_road graph 级在拓扑精确性(TOPO)和路网覆盖(recall)上占优

## 二、partial 采样策略实验

| 训练采样 | 推理 partial | APLS | TOPO F1 | val_loss |
|---|---|:---:|:---:|---|
| edge_random | edge_random | 0.5864 | 0.7724 | 平滑 |
| edge_random | component | 0.6204 | 0.7714 | 平滑 |
| component | component | 0.4571 | 0.6316 | **震荡** |

**结论**:edge_random 训练 + component 推理(0.6204)远好于 component 训练(0.4571)。component 训练每 epoch 整块留/删导致震荡。**最终方案:训练 edge_random(稳定),推理 component(真实不破碎)**。

## 三、踩坑记录

### 坑1:坐标系 bug(completion 比 extraction 差的根因)
- known graph pickle 是 `(row,col)`,render/匹配/注入期待 `(x,y)`,漏了 coord_transform
- 修复:两个入口统一转 `(x,y)`,边端点正确率 0%→100%

### 坑2:component 采样训练震荡
- component 每 epoch 整块留/删,val_loss 0.2~1.5 震荡
- 修复:训练回退 edge_random;推理用 component

### 坑3:方案1 loss(known edge 标签 True)更差
- known edge 标签 True 成捷径监督,recall 降
- 修复:撤销,回退旧 loss

### 坑4:spacenet rn 增量为零
- extraction recall 0.69(强基线),rn 冗余。rn 价值与 extraction 基线强度负相关

### 坑5:partial 可复现性
- 全局 random 累积 → per-file `random.Random(42)`

### 坑6:Dropout 步 valid 未恢复
- dropout 清零先验后 valid 没更新 → 恢复 True

### 坑7:推理 dtype bug
- numpy float64 → torch float32

### 坑8:任务设定理解偏差(traj 角色)
- traj 是训练增强模态,推理不用

### 坑9:xian 黑边不匹配
- 38 个边缘 region sat 有黑边但 rn 有路。cohen-sutherland 裁剪 rn graph 对齐 sat,裁断处补 node

### 坑10:partial PNG thickness 不一致
- partial.png thickness 3→2,对齐 prepare_dataset 和 DelvMap

### 坑11:评测 GT 未裁剪 + 用了 epoch1 ckpt
- **症状**:extraction APLS 0.4288→0.2183 暴跌
- **根因1**:clip_rn_to_sat 只裁了 refine_gt_graph.p(训练GT),没裁 graph_gt.pickle(评测GT) → 黑边 region train/eval GT 不一致
- **根因2**:run.py 默认 `--checkpoint auto` 选了 epoch1(val_loss 最低但欠训练)
- **修复**:① clip_rn_to_sat 也裁 graph_gt.pickle ② 推理用 `--checkpoint epoch:9`
- **结果**:extraction APLS 恢复 0.4211

## 四、最终方案

| 场景 | 采样 | 说明 |
|---|---|---|
| 训练 | edge_random | epoch 间稳定,监督均匀 |
| 推理 | component | 真实不破碎,edge_random 训练可泛化 |
| 评测 GT | graph_gt.pickle 也裁剪 | 和训练 GT 一致(都裁黑边) |
| ckpt 选择 | epoch:9 | 不用 auto/best(val_loss 最低≠APLS 最高) |
