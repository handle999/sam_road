# Partial 路网采样策略改进分析

> **问题**:当前 partial(已知路网)采样按边随机删 50%,导致已知路网约 26 块(didi_xian)/15 块(spacenet),支离破碎,不符合真实地图形态。
>
> **目标**:让 partial 保持路网连续性——真实场景的已知路网应是"几片完整路段 + 缺失区域",而非"每条路都断成碎片"。

---

## 一、现状:当前采样破碎严重

### 1.1 数据集 graph 文件说明(didi_xian)
| 文件 | 含义 |
|------|------|
| `graph_gt.pickle` | OSM 原始图(从 shpfile),连通性好,可能稠密化 |
| `refine_gt_graph.p` | graph_gt 处理版(立交表示更合理)— **训练用 GT** |
| `active_graph.p` | 轨迹匹配图(忽略) |
| `refine_gt_graph_partial.p` | 对 refine_gt_graph 按边随机采样 50% — **当前 partial,破碎** |

> refine_gt_graph 稠密化过(节点密度大,为 node 预测连贯),路段(路口间)很短,按边删必然碎。

### 1.2 实测破碎度(keep 50%,前30张)

| 数据集 | 完整GT块数 | 当前partial块数 | 破碎倍数 |
|--------|:---:|:---:|:---:|
| didi_xian | 2.0 | 26.6 | 13× |
| spacenet | 3.8 | 15.2 | 4× |

## 二、三种候选策略对比

实测三种采样策略(均 keep 50%,didi_xian 前30张):

| 策略 | didi_xian 块数 | spacenet 块数 | 形态 | 真实性 |
|------|:---:|:---:|------|:---:|
| 完整GT(对照) | 2.0 | 3.8 | 自然路网 | — |
| **按边随机(当前)** | 26.6 | 15.2 | 每条路碎成多段 | ❌ |
| 按路段删 | ~26 | ~15 | 仍碎(路段太短) | ❌ 不奏效 |
| **按连通块保** | 1.0 | 2.5 | 整片区域完整,缺另几片 | ✅ 推荐 |
| BFS生长 | 1.0 | 1.0 | 单连通,缺边缘 | ⚠️ 过理想 |

### 2.1 为什么"按路段删"不奏效?
- refine_gt_graph 稠密化,路段(路口间)很短(didi_xian 中位~30px)
- 删 50% 路段仍把连通块拆散(26块),与按边随机无异
- "按路段删"只在**路段长**(路口稀疏)的数据集上有效,didi_xian 不满足

### 2.2 "按连通块保"为什么好?
- 按连通块大小排序,整块保留直到达到 keep_ratio
- 破碎度降到 1.0/2.5 块(接近完整 GT 的 2.0/3.8)
- 形态:"某片区域路网完整,另一片缺失"——符合真实地图(已有区域完整,待测区域缺失)
- 实现简单

### 2.3 "BFS生长"的取舍
- 破碎度最低(1.0),但永远单连通,可能过于理想
- 真实已知路网可能有多个不连通片(不同区域分别已测绘)

---

## 三、推荐方案:按连通块保

### 3.1 算法
```
1. 对完整GT图求连通块, 按大小降序排序
2. 整块保留, 直到累计边数达到 keep_ratio * 总边数
3. 最后一块若超目标, 用 BFS 从该块随机种子生长到目标边数 (保持连通)
4. 输出: known_edges_original (原始图) + known_edge_set_subdivide (细分图对齐)
```

### 3.2 两级对齐(解决预存的不一致 bug)
当前 `_create_known_graph` 在原始图和 subdivided 图上**两次独立随机采样**,导致 GNN 拓扑先验和 render 几何先验不一致。新方案:
- 在原始图按连通块保采样
- subdivided 图按"整条路段的所有细分段一起留/删"对齐
- 消除不一致

### 3.3 应用范围
- `generate_partial_prior.py`(推理用 partial 生成)
- `dataset_completion.py` 的 `_create_known_graph`(训练在线采样)
- 两者用同一策略,保证 train/infer 一致
