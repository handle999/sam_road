# 服务器编排重构手册 (Server Orchestration Refactor Playbook)

> **用途**：把一台服务器上的 `sam_road` 仓库整理到统一的 `run.py` 编排体系：`train → infer → eval` 走同一个 `run_id`，产物集中到 `runs/{run_id}/`，旧 `checkpoints/` 权重可迁移，后续实验可一条命令复现。
>
> **适用对象**：服务器上已有自己的权重、推理产物、日志、`.gitignore` 的仓库。服务器状态不能照搬本地假设，必须先探测再操作。
>
> **当前基线**：本手册按最近编排体系改造后的代码编写，要求仓库已有 `run.py`、`tools/registry.py`、`tools/config_utils.py`、`tools/migrate_ckpts_to_runs.py`、`tools/server_preflight.py`，且 6 个 train/infer 脚本已支持 `--run-root`。

## 1. 总原则

1. **先自检，后操作**：先运行 `python tools/server_preflight.py`，再决定迁移 / 清理 / 训练。
2. **不照搬时间戳**：服务器上的 ckpt / run_info / save 目录时间戳可能和本地不同，一律以探测结果为准。
3. **不删除 ckpt**：ckpt 是昂贵产物；只能移动/备份，不能随手 `rm`。
4. **不提交大产物**：`.ckpt`、推理 graph/mask/viz、逐图 metric 明细不进 git；`profile.yaml`、`run_info.yaml`、`best_ckpt.txt`、汇总 json 可以进 git。
5. **数据集名统一**：注册表 key 是 `spacenet` / `didi_xian` / `cityscale`，不是 `xian`；历史 `didi` 只作为 metric 兼容别名。

## 2. 开始前一键自检

在服务器仓库根目录执行：

```bash
cd /home/hanhaoyu/sam_road   # 或服务器实际项目根
python tools/server_preflight.py | tee /tmp/samroad_preflight.txt
```

重点看 7 块输出：

1. **编排代码是否就位**：`run.py` / registry / migrate / batch 是否存在，6 个脚本是否都含 `--run-root`。
2. **旧权重探测**：`checkpoints/` 下是否还有 `.ckpt`。
3. **旧推理产物**：`save/` 是否还有历史 infer 输出。
4. **runs/ 现状**：是否已有 profile / run_info / ckpt。
5. **迁移需求判断**：脚本会提示情况 A/B/无权重。
6. **.gitignore 关键规则**：是否已有 `runs/` 规则和 `*.ckpt` 忽略。
7. **GPU / git 状态**：服务器是否能跑 CUDA，工作区是否干净。

如果脚本提示编排代码缺失：

```bash
git pull origin main
python tools/server_preflight.py
```

## 3. 目标目录结构

重构后，每个实验一个 `run_id`：

```text
runs/{run_id}/
├── profile.yaml             # 进 git: 实验声明、config snapshot、git/env 信息
├── best_ckpt.txt            # 进 git: 当前 best ckpt 路径
├── .step_done/
│   ├── train                # 进 git: 步骤完成标记
│   ├── infer
│   └── eval
├── train/
│   ├── checkpoints/         # 不进 git: *.ckpt
│   ├── log.txt              # 不进 git: 训练全文日志
│   ├── run_info_*.yaml      # 进 git: 训练执行记录
│   └── csv/                 # 不进 git
├── infer/
│   ├── config.yaml          # 进 git: 推理 config 副本
│   ├── run_info.yaml        # 进 git: 推理命令/ckpt/git/env
│   ├── inference_time.txt   # 进 git: 推理耗时
│   ├── graph/ mask/ viz/    # 不进 git: 推理大产物
└── eval/
    ├── apls.json            # 进 git: 汇总 metric
    ├── topo.json            # 进 git: 汇总 metric
    ├── apls/ topo/          # 不进 git: 逐图明细
```

`run_id` 默认格式：

```text
{task}_{dataset}_{timestamp}
# 例: completion_didi_xian_20260617_194853
```

## 4. 迁移旧 checkpoints 到 runs/

### 4.1 判断是否需要迁移

```bash
if [ -d checkpoints ] && [ "$(find checkpoints -name '*.ckpt' 2>/dev/null | head -1)" ]; then
  echo "情况 A: 旧权重在 checkpoints/, 需要迁移"
elif [ -d runs ] && [ "$(find runs -name '*.ckpt' 2>/dev/null | head -1)" ]; then
  echo "情况 B: ckpt 已在 runs/, 无需迁移"
else
  echo "无本地 ckpt，可能只有 profile metadata，后续可直接训练或从备份补 ckpt"
fi
```

注意：仓库里可能已经有 `runs/*/profile.yaml`，但因为 `.ckpt` 被忽略，服务器本地未必有真实 ckpt。必须用 `find runs -name '*.ckpt'` 核实。

### 4.2 dry-run 预览

```bash
python tools/migrate_ckpts_to_runs.py --dry-run
```

核对：

- `samroad_spacenet` → `extraction_spacenet_<ts>`
- `samroad_didi_xian` → `extraction_didi_xian_<ts>`
- `samroad_completion_spacenet` 或历史 `samroad_completion` → `completion_spacenet_<ts>`
- `samroad_completion_didi_xian` → `completion_didi_xian_<ts>`
- `samroad_4ch_*` → `4ch_*_<ts>`

如果 dry-run 识别不了某目录，先不要执行，手动看：

```bash
ls checkpoints/<unknown_dir>/
cat checkpoints/<unknown_dir>/run_info_*.yaml | grep -E "script:|config_source:|model:"
```

### 4.3 执行迁移

```bash
python tools/migrate_ckpts_to_runs.py --execute
```

验证：

```bash
find runs -maxdepth 3 -type f | sort | head -80
find runs -name '*.ckpt' | sort | head
python - <<'PY'
import os, sys
sys.path.insert(0,'.')
from tools.config_utils import get_best_ckpt
for rid in sorted(os.listdir('runs')):
    if not os.path.isdir(f'runs/{rid}'):
        continue
    try:
        b = get_best_ckpt(rid)
        print(f'{rid}: {b}')
    except Exception as e:
        print(f'{rid}: <no ckpt> {e}')
PY
```

### 4.4 手动迁移非标准目录

仅当迁移脚本识别不了目录时使用。示例：

```bash
TASK=completion
DATASET=spacenet
TS=20260620_100000
RUN_ID=${TASK}_${DATASET}_${TS}

mkdir -p runs/${RUN_ID}/train/checkpoints
mv checkpoints/my_custom_dir/*.ckpt runs/${RUN_ID}/train/checkpoints/
mv checkpoints/my_custom_dir/run_info_*.yaml runs/${RUN_ID}/train/ 2>/dev/null || true

python - <<PY
import os, yaml, sys
sys.path.insert(0,'.')
from tools.config_utils import write_profile, mark_step_done, save_config_snapshot
from tools.run_info import _try_get_git
rid = '${RUN_ID}'
run_info_files = [f for f in os.listdir(f'runs/{rid}/train') if f.startswith('run_info')]
config_source = None
if run_info_files:
    with open(f'runs/{rid}/train/{run_info_files[0]}') as f:
        config_source = yaml.safe_load(f).get('config_source')
profile = {
    'run_id': rid,
    'task': '${TASK}',
    'dataset': '${DATASET}',
    'created_at': '${TS}',
    'migrated_from': 'checkpoints/my_custom_dir',
    'config_source': config_source,
    'config_snapshot': save_config_snapshot(config_source, rid) if config_source else None,
    'steps_requested': ['train','infer','eval'],
    'git': _try_get_git('.'),
    'step_status': {'train': 'done (manual migration)'},
}
write_profile(rid, profile)
mark_step_done(rid, 'train')
print('manual migration done:', rid)
PY
```

## 5. .gitignore 适配

目标：`runs/` 中 metadata 进 git，大产物不进 git。

检查：

```bash
grep -nE "runs|ckpt|checkpoints|save|train_logs" .gitignore
```

应至少包含：

```gitignore
# Model checkpoints and outputs
checkpoints
save
*.ckpt

# runs/ 统一实验目录: 保留溯源 yaml/json, 忽略大产物 (ckpt 已被 *.ckpt 忽略)
runs/*/*/csv/
runs/*/infer/mask/
runs/*/infer/viz/
runs/*/infer/graph/
runs/*/eval/apls/
runs/*/eval/topo/
runs/*/train/csv/
!runs/**/*.yaml
!runs/**/*.json
```

验证：

```bash
# ckpt 应被忽略
git check-ignore runs/*/train/checkpoints/*.ckpt 2>/dev/null && echo "✓ ckpt ignored"

# metadata 应可被 git add
git add -n runs/ 2>&1 | grep -E "profile.yaml|run_info|best_ckpt|apls.json|topo.json" | head
```

如果服务器 `.gitignore` 有自己的规则，不要删，只追加 `runs/` 规则。若已有 `runs/` 整体忽略，需要增加 `!runs/**/*.yaml` / `!runs/**/*.json` 例外。

## 6. 端到端验证

### 6.1 dry-run 验证四个默认组合

```bash
for combo in "extraction spacenet" "completion spacenet" "extraction didi_xian" "completion didi_xian"; do
  set -- $combo
  echo "=== $1 × $2 ==="
  python run.py --task $1 --dataset $2 --steps train --gpus 0 --dry-run
  echo
done
```

确认 config 都存在：

```bash
python - <<'PY'
import os, sys
sys.path.insert(0,'.')
from tools.registry import default_config_for
for t in ['extraction','completion','4ch']:
    for d in ['spacenet','didi_xian']:
        p = default_config_for(t,d)
        print(('✓' if os.path.exists(p) else '✗'), t, d, p)
PY
```

### 6.2 使用已有 ckpt 只跑 infer+eval

```bash
RUN_ID=$(ls runs | grep completion_spacenet | head -1)
python run.py --task completion --dataset spacenet \
  --run-id "$RUN_ID" \
  --steps infer,eval \
  --resume-run \
  --checkpoint auto \
  --gpus 0 \
  --workers 8
```

预期：

- `--checkpoint auto` 会从 `runs/{run_id}/train/checkpoints/` 自动选 `val_loss` 最低 ckpt。
- 输出写入 `runs/{run_id}/infer/` 和 `runs/{run_id}/eval/`。
- 完成后 `.step_done/infer`、`.step_done/eval` 会出现。

### 6.3 Completion no-prior / full-prior 对照

默认 completion 会从 registry 注入已知图路径：

- SpaceNet：`datasets/spacenet/RGB_1.0_meter`
- DiDi Xian：`datasets/didi/xian/2019_400/xian_2019_400`，并附带 `--traj_dir`。

若要纯 extraction fallback：

```bash
python run.py --task completion --dataset didi_xian \
  --run-id completion_didi_xian_no_prior \
  --steps infer,eval \
  --checkpoint auto \
  --no-input-graph \
  --gpus 1 \
  --workers 8
```

- `runs/*/profile.yaml`

- `runs/*/.step_done/*`

## 9. 故障排查

| 症状 | 排查 / 修复 |
|------|-------------|
| `run.py` 找不到 config | `python - <<'PY' ... default_config_for(...)` 检查 registry；必要时 `--config` 显式覆盖。|
| dataset 名报错 | 只用 `spacenet` / `didi_xian` / `cityscale`。旧 `xian`、`didi` 不作为编排 key。|
| `--checkpoint auto` 选不到 ckpt | 确认 `runs/{run_id}/train/checkpoints/*.ckpt` 真实存在；只有 metadata 没有 ckpt 时需从服务器备份补大文件。|
| best ckpt 不对 | `python -c "from tools.config_utils import select_best_ckpt; print(select_best_ckpt('<run_id>'))"`。文件名需含 `val_loss=`；否则可能退到 `last.ckpt`。|
| 推理 dtype 报 `Float/Double` | `grep "batch_points = torch.tensor" engine/inferencer_completion.py`，应显式 `dtype=torch.float32`。若没有，说明代码没更新。|
| GPU 不可用 | `nvidia-smi` + `python -c "import torch; print(torch.cuda.is_available())"`。如果 NVML 报某张卡 Unknown Error，需管理员重启驱动。|
| eval 的 APLS 0.0 | 确认 `--dir` 是相对项目根，如 `save/xxx` 或 `runs/<id>/infer`；现在脚本会 normalize 并在全 skip 时报警。|
| `didi_xian` eval 不认 | 需要包含 `fix(metrics): standardize Xian dataset name to didi_xian` 之后的代码；`python metrics/eval.py --help` 应显示 `didi_xian`。|
| ckpt 误进 git | `.gitignore` 缺 `*.ckpt` 或 `git add -f` 误操作。用 `git rm --cached <file.ckpt>` 移出索引，再补 ignore。|
| 迁移脚本不识别旧目录 | 旧目录名不符合 `samroad_{variant}_{dataset}` 规则，按 §4.4 手动迁移。|

## 10. 给服务器 Claude 的操作约束

1. 每一步都先读现状：`tools/server_preflight.py`、`git status`、`find checkpoints/runs`。
2. 不删除 ckpt，只移动/备份。
3. 不把 ckpt、graph、mask、viz、逐图 metric 明细提交到 git。
4. 不使用旧 dataset 名 `xian`；统一 `didi_xian`。
5. 不照搬本地 run_id / 时间戳；服务器 run_id 以服务器自己的 `run_info` 和 `profile` 为准。
6. 如果发现手册与 `python run.py --help` 不一致，以代码为准，并同步更新本手册。
